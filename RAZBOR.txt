1. heroui - библиотека компонентов.
2. config - конафигурационная папка, в ней храним повторяющиеся данные, вычисляемые значния и т.п. Внутри создаем сколько угодно файлов, название должно состоять из name.config.ts. Name - это уникальное название по теме данных, например, если это данные для layout, то layout.config.ts, если общие данные сайта, то site.config.ts. Внутри файлов создаем и экспортируем объект, в который записываем нужные значения.
3. prisma orm - современный ORM (Object-Relational Mapping) для работы с базами данных. Упрощает взаимодействие с БД (PostgreSQL, MySQL, SQLite, MongoDB, SQL Server и др.), предоставляя удобный и безопасный API.
4. PostgreSQL (или просто Postgres) — это сервер базы данных (СУБД), который хранит, управляет и обрабатывает данные. В этом проекте использовался как локальная база данных.
5. pgAdmin - графическая программа для управления базами, это просто удобный интерфейс для работы с БД в PostgreSQL.
6. Auth.js - библиотека для аутентификации для Next. В проекте используем JWT способ авторизации - храним в куках токен содержащие данные пользователя, в этом случае передаем данные между клиентом и сервером без запросов к БД.
7. zod - библиотека для валидации введенных данных авторизации. Установив, добовляем файл со схемой валидации, где идет проверка на длину прочее, смотря что нужно.
8. bcryptjs - библиотека для сравнения паролей, так как пароли хранятся в базе данных захешированными, то есть не в обычном виде, то и сравнить их просто так не получится, для этого нужна специальная библиотека.
9. Zustand — библиотека для управления состоянием, альтернатива Redux, но без лишнего шаблонного кода, с простым и понятным API, без провайдеров и контекста.
10.  DOMPurify - библиотека для санитации - это процесс очистки HTML кода от потенциально опасных элемнтов и аттрибутов. В проекте текстовый контент для страниц пишется в одном файле и каждый контент привязан к определенному урлу, после создался компонент, который вычисляет на каком урле находится пользователь и отдает нужный контент. Проблема в том, что текст написан в `` и там же размечен, а значит отображается разметка как текст. Для этого был применен аттрибут <div dangerouslySetInnerHTML={{ __html: pageContent.content }}></div> - который разрешает добавлять в разметку все что угодно, а то максимально опасно. Используя библиотеку, деаем использование dangerouslySetInnerHTML безопасным. npm i dompurify @types/dompurify.
11. HTML React Parser — библиотека, преобразует HTML-строку в элементы React для рендеринга в приложении. Она позволяет вставлять HTML-контент из внешних источников (например, из API или CMS) внутрь приложения. В проекте была установлена, чтобы заменить опасный dangerouslySetInnerHTML, при этом DOMPurify все равно используется.
